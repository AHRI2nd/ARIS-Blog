Lyrical SyncのAI自動同期機能は、CTC Forced Alignerを使って歌詞テキストにタイムスタンプを自動付与します。  
オプションでDemucsによるボーカル分離を前処理として使用し、精度を向上させることができます。

---

## システム要件

| 項目 | 最小仕様 |
|------|---------|
| OS | macOS 12+, Windows 10+ |
| RAM | 8 GB（16 GB 推奨） |
| ストレージ | 必須モデルのみ: 約3 GB / フル: 約5 GB |
| インターネット | 初回セットアップ時に必要 |

> [!NOTE]
> **GPUアクセラレーション**: 現在のバージョンはCPU推論を使用します。パフォーマンスはCPUの速度に依存します。

---

## ステップ1 — Python環境のインストール

AI機能はアプリ内蔵Python（CPython 3.11）を使用するため、別途Pythonのインストールは不要です。

1. アプリ左下の **⚙ 設定** ボタンをクリックします。
2. **AI環境** タブを選択します。
3. **「Pythonをダウンロード」** ボタンをクリックします。
   - macOS: `python-3.11.10-macos-...` ディストリビューションがダウンロードされます。
   - Windows: `python-3.11.10-windows-...` ディストリビューションがダウンロードされます。
4. ダウンロード完了後、**「パッケージをインストール」** ボタンをクリックします。
   - `torch`、`unidecode`、`demucs`、`ctc-forced-aligner` などがインストールされます。
   - Windowsでは `ctc-forced-aligner` をソースからビルドします（10〜20分かかります）。
5. インストールログに **「インストール完了」** と表示されたら完了です。

> [!WARNING]
> **Windowsの注意**: Visual Studio Build Tools（C++ビルドツール）がインストールされていない場合、ソースビルドが失敗する可能性があります。エラーが発生した場合は、先に [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) をインストールしてください。

---

## ステップ2 — モデルのダウンロード

1. 設定の **AIモデル** タブを選択します。
2. 以下の表を参考に、必要なモデルをダウンロードします。

| モデル | 用途 | サイズ | 必須 |
|--------|------|--------|------|
| **ctc-mms-300m** | タイムスタンプ強制アライメント（1,100以上の言語対応） | 1.2 GB | ✅ 必須 |
| demucs-htdemucs | ボーカル/伴奏分離の前処理 | 83 MB | 任意 |
| wav2vec2-base-960h | 英語ASR補助モデル | 380 MB | 任意 |

3. モデルファイルはデフォルトでアプリデータディレクトリに保存されます。  
   カスタムパスを指定するには、**「モデル保存パス」** から変更できます。

> [!TIP]
> まずctc-mms-300mだけでお試しください。Demucsは伴奏が多い曲で精度を高めますが、処理時間が追加されます。

---

## ステップ3 — AI自動同期の実行

1. オーディオファイルを開きます（MP3、WAV、FLAC、M4Aなど）。
2. LRCエディターに歌詞を入力します。
3. エディター上部の **「AI自動同期」** ボタンをクリックします。
4. 言語を選択します。
   - 韓国語: `kor`
   - 英語: `eng`
   - 日本語: `jpn`
   - その他1,100以上の言語に対応（ISO 639-3コード）
5. **「実行」** ボタンをクリックします。

### 進行状態

| 状態 | 説明 |
|------|------|
| モデル読み込み中 | ctc-mms-300mをメモリに展開中 |
| ボーカル分離 | Demucsでボーカルトラックを抽出中（Demucsインストール時） |
| アライメント分析 | 歌詞とオーディオをマッチング中 |
| 完了 | タイムスタンプ生成完了 |

---

## ステップ4 — 結果の確認と修正

- 各歌詞行の横に **信頼度バッジ**（0〜100%）が表示されます。
- 信頼度が低い行は赤いバッジで表示されます。
- 修正方法:
  - 該当行をクリックしてアクティブにし、再生中に **Space** キーで手動スタンプ
  - タイムスタンプボタンを直接クリックして現在の再生位置で上書き
- 手動スタンプした行の信頼度バッジは自動的に削除されます。
- 確認が終わったら **「下書きをクリア」** ボタンで信頼度バッジを一括削除できます。

## 設定オプション

| オプション | 場所 | 説明 |
|-----------|------|------|
| 空行オフセット | 設定 → 一般 | 空行（段落区切り）のタイムスタンプに適用するオフセット秒数（デフォルト: 1.0秒） |
| モデル保存パス | 設定 → AIモデル | デフォルトパスの代わりにカスタムパスにモデルを保存 |

## よくある質問

**Q. アライメントがずれている行が多いです。**  
A. 歌詞に誤字や不要な記号（括弧、コーラス表記など）がないか確認してください。伴奏が多い曲はDemucsをインストールして再試行すると精度が上がります。

**Q. 「Pythonの準備ができていません」というエラーが出ます。**  
A. 設定 → AI環境でPythonのインストール状態を確認してください。インストール済みでもエラーが続く場合は、アプリを再起動してみてください。

**Q. macOSで「破損したファイル」の警告が出ます。**  
A. アプリが自動的にquarantine属性を削除しますが、失敗した場合はターミナルで以下のコマンドを実行してください。

```bash
xattr -rd com.apple.quarantine ~/Library/Application\ Support/lyrical-sync/python
```

**Q. Windowsでパッケージのインストールが失敗します。**  
A. Visual Studio Build Toolsの「C++によるデスクトップ開発」ワークロードをインストールしてから再試行してください。
